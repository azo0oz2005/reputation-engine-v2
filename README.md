# سمعة برو

خدمة SaaS عربية بسيطة لأصحاب الأنشطة المحلية (الكافيهات، المطاعم، العيادات، الصالونات، مغاسل السيارات، ومراكز الصيانة) لإدارة سمعتهم على Google.

## الفكرة

أغلب العملاء الراضين يخرجون دون أن يقيّموا النشاط على Google، بينما العميل غير الراضي غالبًا يكتب تقييمًا سلبيًا علنيًا. سمعة برو تعطي النشاط رمز QR ذكي:

- إذا كان العميل **راضيًا** (٤ أو ٥ نجوم) يتم توجيهه إلى صفحة تقييم Google الخاصة بالنشاط.
- إذا كان العميل **غير راضٍ** (١ إلى ٣ نجوم) يتم توجيهه إلى نموذج ملاحظة داخلي تصل صاحب النشاط مباشرة لتحسين التجربة.

الخدمة تساعدك على:
1. جمع تقييمات Google من العملاء الراضين بسهولة عبر QR Code.
2. استقبال ملاحظات العملاء غير الراضين داخليًا.
3. متابعة رضا العملاء من لوحة تحكم بسيطة.
4. معرفة عدد العملاء الذين ضغطوا على رابط تقييم Google.
5. معرفة الشكاوى والملاحظات المتكررة.

## التقنيات

- Python + Flask
- Flask-SQLAlchemy + SQLite
- qrcode + Pillow (لتوليد رموز QR)
- python-dotenv
- gunicorn (للنشر)
- HTML Templates + Tailwind CSS (عبر CDN) + خط Cairo

## التشغيل محليًا

```bash
# 1) إنشاء البيئة الافتراضية
python -m venv venv

# 2) تفعيل البيئة (Windows)
venv\Scripts\activate

# على macOS / Linux:
# source venv/bin/activate

# 3) تثبيت المتطلبات
pip install -r requirements.txt

# 4) (اختياري) إنشاء ملف البيئة
copy .env.example .env   # على Windows
# cp .env.example .env   # على macOS / Linux

# 5) تشغيل التطبيق
python app.py
```

ثم افتح المتصفح على: http://127.0.0.1:5000

> قاعدة البيانات تُنشأ تلقائيًا عند أول تشغيل داخل مجلد `instance/`.

## طريقة الاستخدام

### إضافة نشاط
1. افتح الصفحة الرئيسية واضغط **«إضافة نشاط جديد»** أو انتقل إلى `/admin/add`.
2. أدخل اسم النشاط، نوعه، رابط تقييم Google Maps، ورقم الجوال/الإيميل (اختياري).
3. يمكنك ترك حقل المعرّف (slug) فارغًا ليُنشأ تلقائيًا من اسم النشاط.
4. بعد الحفظ يتم توليد رمز QR وتحويلك مباشرة إلى لوحة التحكم.

### تجربة QR
1. من لوحة التحكم، حمّل رمز QR أو انسخ رابط صفحة التقييم `/r/<slug>`.
2. امسح الرمز بكاميرا الجوال (أو افتح الرابط مباشرة).
3. اختر تقييمًا من ١ إلى ٥ نجوم:
   - ٤ أو ٥ نجوم → صفحة شكر وزر «قيّمنا على Google».
   - ١ إلى ٣ نجوم → نموذج ملاحظة داخلي.
4. ارجع للوحة التحكم لرؤية الإحصائيات والملاحظات الواردة.

## المسارات

| المسار | الوصف |
|--------|-------|
| `/` | الصفحة الرئيسية |
| `/admin/add` | إضافة نشاط جديد |
| `/r/<slug>` | صفحة تقييم العميل (QR) |
| `/happy/<feedback_id>` | صفحة العميل الراضي |
| `/go-google/<feedback_id>` | تحويل العميل إلى تقييم Google |
| `/complaint/<feedback_id>` | نموذج ملاحظة العميل غير الراضي |
| `/thank-you` | صفحة شكر |
| `/dashboard/<slug>` | لوحة تحكم صاحب النشاط |

## النشر على Render

1. ارفع المشروع إلى مستودع Git.
2. أنشئ خدمة **Web Service** جديدة على [Render](https://render.com) واربطها بالمستودع.
3. الإعدادات:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `gunicorn app:app`
4. أضف متغيرات البيئة من قسم Environment:
   - `SECRET_KEY` = قيمة سرية قوية.
   - `DATABASE_URL` = `sqlite:///database.db` (أو رابط قاعدة بيانات خارجية).
5. ملاحظة: قاعدة بيانات SQLite على Render غير دائمة بين عمليات النشر؛ للإنتاج الحقيقي استخدم قاعدة بيانات خارجية (مثل PostgreSQL).
